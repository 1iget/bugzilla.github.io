How To Get Bugzilla

Instructions on how to obtain, install and configure Bugzilla are in the Installation and Maintenance Guide. That document also explains how to upgrade Bugzilla to a new version, move a Bugzilla installation between machines, and migrate from other bug-tracking systems.

If you are installing a new Bugzilla, the latest stable release series (and so the one you should choose unless you have a good reason for doing otherwise) is the [% release.stable.branch %] series. You will need this information when following the installation instructions.

If you are upgrading, note that you can upgrade directly from any old Bugzilla version to any new release—you do not have to go via the releases in between. You should upgrade to the latest stable series, [% release.stable.branch %], and read the Release Notes. If you want to know exactly what's changed from one version to the next, you can view the changelogs.

Bugzilla is licensed under the Mozilla Public License 2.0, and is thus both free and open source software.

Stable Release Candidate ([% release.dev.current %])

Bugzilla [% release.dev.current %] is our latest release candidate for Bugzilla [% release.dev.branch %].

Release candidates receive full QA testing, and should be considerably more stable than development releases. They have not been thoroughly tested in the real world, but should be safe for most production environments, particularly small ones.

Note, however, that many Bugzilla extensions (such as Testopia) do not support Release Candidates.

If you do use a Release Candidate, make sure to upgrade to the final release when it comes out.

A look at the new features in Bugzilla [% release.dev.current %] can be found in the Release Notes. Also make sure to read the Release Notes.

[% INCLUDE dl mode = 'rc' release = release.dev %] [% ELSIF release.dev %]

Development Snapshot ([% release.dev.current %])

Bugzilla [% release.dev.current %] is our latest development snapshot leading to Bugzilla [% release.dev.branch %].

Use of a development branch is at your own risk. They receive very little testing, so expect this release to be unstable. Users of [% release.stable.branch %] or older [%+ release.dev.devbranch %] releases are encouraged to try out new [%+ release.dev.devbranch %] releases in a testing environment and let us know of any problems that are found with the release.

A look at the new features that will be in Bugzilla [% release.dev.branch %] can be found in our latest Status Updates.

Old Stable Release ([% release.old.current %])

[% release.old.current %] is the old stable version of Bugzilla. If you are doing a new installation of Bugzilla, we recommend that you use the latest stable version instead of a [% release.old.branch %] version.

An overview of the new features in [% release.old.branch %] can be found in the [% release.old.branch %] Release Notes.

We encourage all users, especially upgraders, to read the Release Notes before installing Bugzilla [% release.old.current %].

Legacy Release ([% release.legacy.current %])

Bugzilla [% release.legacy.current %] is the legacy release of Bugzilla. It is kept here for sites still running the [% release.legacy.branch %] codebase to enable them to pick up security updates. All users are recommended to use Bugzilla [% release.stable.current %] instead, and users of Bugzilla [% release.legacy.branch %] are encouraged to upgrade to [% release.stable.current %] as soon as possible.
